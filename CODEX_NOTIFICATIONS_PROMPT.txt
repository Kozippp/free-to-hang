   - Get Expo project ID from app.json
   - Test with Expo Go first, then production build

   - Simple: friend goes online → all their friends get notified
   - No complex algorithms for MVP
   - Just notify immediately

5. RE-ENGAGEMENT
   - Check daily at 6 PM
   - Only notify if inactive 3+ days AND have active friends
   - Use node-cron scheduler

6. REAL-TIME UPDATES
   - Use Supabase Realtime for instant notification updates
   - Subscribe to notifications table filtered by user_id
   - Handle INSERT, UPDATE, DELETE events

7. BADGE COUNT
   - Update badge on tab bar
   - Show unread count from notifications store
   - Clear when user opens notifications

8. NAVIGATION
   - Each notification type has specific navigation
   - Use notification.data to pass plan_id, user_id, etc.
   - Handle deep linking from push notifications
